# 要件定義

**プロジェクト名**: 戦略コンサルタント提案作成ツール
**作成日**: 2025-12-07
**対象ユーザー**: 戦略コンサルタント（自分自身 + チームメンバー）

---

## 📋 プロジェクト概要

戦略コンサルタントが作成する提案書（パワーポイント）を、対話形式での言語化サポートを通じて自動生成するツール。

### 目的
- 提案書作成の効率化（言語化→構成→スライド化の一貫した流れ）
- ロジカルで美しい提案書の品質担保（1スライド1メッセージ、MECEの原則）
- クライアント課題の深掘りと仮説立案のサポート

### 解決したい課題
- 提案書作成に時間がかかりすぎる
- 骨子の言語化とスライド化が分断されている
- 必須要素の抜け漏れが発生する
- チームメンバー間での品質のばらつき

---

## 👤 ユーザーストーリー

### US-1: 対話形式での骨子作成（現状認識）

**As a** 戦略コンサルタント
**I want** クライアントの現状認識（背景、問題、原因仮説）を対話形式で言語化したい
**So that** 提案の前提となる課題を正確に把握できる

#### 受け入れ基準
- [ ] クライアント情報（組織、業界、背景）を入力できる
- [ ] 「顕在化している問題」を対話形式で深掘りできる
- [ ] 「問題の原因仮説」をAIと対話しながら明確化できる
- [ ] 情報が不足している場合、AIから追加質問を受け取れる
- [ ] 背景のレイヤー（業界/全社/事業部等）を選択できる
- [ ] 言語化した内容を後で編集・修正できる

#### 優先度
- **必須**（MVP）

---

### US-2: 対話形式での骨子作成（課題仮説）

**As a** 戦略コンサルタント
**I want** 原因仮説から導かれるクリティカルな課題を言語化したい
**So that** 提案の核となる課題設定を明確にできる

#### 受け入れ基準
- [ ] 原因仮説を基に、課題候補をAIが提示する
- [ ] 複数の課題候補からクリティカルなものを選択できる
- [ ] 課題設定が不十分な場合、AIから深掘り質問を受け取れる
- [ ] イシューツリー形式での整理が可能（任意要素）
- [ ] MECE性を意識した構造化ができる

#### 優先度
- **必須**（MVP）

---

### US-3: 対話形式での骨子作成（ToBe像とアプローチ）

**As a** 戦略コンサルタント
**I want** 将来の目指すべき姿（ToBe）とアプローチ方針を言語化したい
**So that** 提案の方向性とスコープを明確にできる

#### 受け入れ基準
- [ ] 課題を解決した先の「ToBe像」を対話形式で言語化できる
- [ ] ToBeと現状のギャップから、プロジェクトスコープを設定できる
- [ ] アプローチの全体像とステップ詳細を言語化できる
- [ ] 各ステップの概要を入力できる
- [ ] 言語化内容を前のステップに戻って修正できる

#### 優先度
- **必須**（MVP）

---

### US-4: 必須スライド要素の選択・生成

**As a** 戦略コンサルタント
**I want** 提案書に必要なスライド要素を選択・自動生成したい
**So that** 抜け漏れなく、標準的な提案書を作成できる

#### 受け入れ基準
- [ ] 以下の必須要素が自動で含まれる：
  - エグゼクティブサマリー
  - 現状認識と課題仮説
  - ToBe像
  - プロジェクトゴールとアプローチ方針
  - アプローチ概要
  - アプローチ詳細（各ステップ）
  - スケジュール
  - 体制
  - 見積もり
  - 見積もり前提
- [ ] 以下の任意要素を選択できる：
  - イシューツリー
  - なぜこのアプローチか
  - 会議体
  - PJTメンバー
  - Appendix
- [ ] 各スライドに「1スライド1メッセージ」の原則でメインメッセージが自動生成される
- [ ] スライドの順序を調整できる

#### 優先度
- **必須**（MVP）

---

### US-5: パワーポイント形式でのエクスポート

**As a** 戦略コンサルタント
**I want** 作成した提案書をパワーポイント形式でエクスポートしたい
**So that** クライアントに提出できる形式で保存できる

#### 受け入れ基準
- [ ] 生成したスライドをPPTX形式でダウンロードできる
- [ ] スライドのデザインが「綺麗でロジカル」な基準を満たす：
  - コーポレートカラー2色 + 黒/グレーのみ使用
  - 強調部分のみカラー使用
  - 情報量が多くてもメッセージがシンプルに伝わる
- [ ] 指定したテンプレート・カラー・フォントが適用される（任意）
- [ ] エクスポート前にプレビュー表示できる

#### 優先度
- **必須**（MVP）

---

### US-6: 前ステップへの戻り機能

**As a** 戦略コンサルタント
**I want** スライド生成後も前のステップに戻って修正したい
**So that** イメージに合わない場合に柔軟に調整できる

#### 受け入れ基準
- [ ] スライド選択画面から骨子作成画面に戻れる
- [ ] 骨子を修正すると、関連スライドが自動更新される
- [ ] 編集履歴が保存され、過去の状態に戻せる（任意）

#### 優先度
- **高**（MVP後の優先機能）

---

### US-7: データの複数デバイス対応

**As a** 戦略コンサルタント
**I want** PCとiPadの両方で提案書を編集・確認したい
**So that** 場所を選ばずに作業できる

#### 受け入れ基準
- [ ] クラウドDBにデータが保存される（Supabase推奨）
- [ ] 異なるデバイスで同じデータにアクセスできる
- [ ] ログイン機能がある（メールアドレス認証）
- [ ] データの自動保存機能がある

#### 優先度
- **高**（MVP後の優先機能）

---

### US-8: チームメンバー情報の管理

**As a** コンサルティングファームのマネージャー
**I want** 自社メンバーのスキル・実績・単価を管理したい
**So that** 適切なメンバーアサインと見積もり計算ができる

#### 受け入れ基準
- [ ] メンバー情報（役職、得意領域、過去実績、単価）を登録できる
- [ ] 必要スキルに基づいてメンバー候補が提示される
- [ ] メンバーをプロジェクトにアサインできる
- [ ] メンバー情報を編集・削除できる

#### 優先度
- **中**（将来機能 - US-9の前提）

---

### US-9: 見積もり根拠の自動生成

**As a** 戦略コンサルタント
**I want** スケジュールと体制から見積もりを自動計算したい
**So that** 工数と単価に基づいた正確な見積もりを提示できる

#### 受け入れ基準
- [ ] 週次スケジュールでアプローチステップが可視化される
- [ ] 各ステップの工数（人日）を入力できる
- [ ] アサインされたメンバーの単価が自動適用される
- [ ] 工数 × 単価で見積額が自動計算される
- [ ] Excel/スプレッドシート形式でエクスポートできる
- [ ] 数式ベースで、工数や単価の変更が見積額に自動反映される

#### 依存関係
- **US-8（チームメンバー情報の管理）が前提**

#### 優先度
- **中**（将来機能 - US-8の後に実装）

---

### US-10: 将来的な拡張性（他フォーマット対応）

**As a** 戦略コンサルタント
**I want** 提案書以外のフォーマット（研修資料等）にも対応したい
**So that** 様々な用途で活用できる

#### 受け入れ基準
- [ ] スライド要素のテンプレートを切り替えられる
- [ ] カスタムテンプレートを追加できる

#### 優先度
- **低**（将来機能）

## 🚫 非機能要件

### パフォーマンス
- ページ読み込み: 3秒以内
- AI対話のレスポンス: 5秒以内
- PPTXエクスポート: 10秒以内（10スライド想定）

### セキュリティ
- クライアント情報は暗号化して保存
- ログイン認証（メールアドレス + パスワード）
- チーム内でのデータ共有権限管理（将来機能）

### 互換性
- 対応ブラウザ: Chrome, Safari, Edge（最新版）
- 対応デバイス: PC, iPad
- PowerPoint互換: Microsoft PowerPoint 2016以降

### ユーザビリティ
- 対話形式のUIは直感的でわかりやすい
- モバイル（iPad）でも操作しやすいUI
- ステップ間の遷移がスムーズ
